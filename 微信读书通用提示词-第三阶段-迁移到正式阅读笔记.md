# 第三阶段：迁移到正式阅读笔记

只有当用户明确说“把已优化内容更新到正式阅读笔记”“整理并更新到阅读笔记里”“更新正式阅读笔记”这类话时，才执行这一阶段。若用户只是要整理、评价、优化中间稿，停在前面阶段，不要提前迁移。

执行口令模板只作为本阶段触发后的工作说明：

```text
请把 `{中间稿路径}` 中已经优化完成、适合沉淀的内容，整理并更新到 `{正式笔记路径}`。
```

迁移时必须同时遵守以下原则：

1. 不要机械搬运所有中间稿，要按材料价值分类安置。
2. 默认优先迁移本轮已优化内容，同时检查前几轮已优化但尚未迁移的遗留内容。有价值的应一并迁移或列入待回看，不要默认遗漏。
3. 每条迁移材料必须先保留书籍划线原文，再保留“我自己写的内容”。不要用 AI修正覆盖原文。
4. 所有迁移卡按“划线原文 / 我自己写的内容 / AI评价 / AI修正 / AI补充”的顺序归档。
5. 轻卡的 AI 部分可以简短，但不能丢掉“划线原文 + 我自己写的内容”这一层。
6. 值得保留的外部读者原话优先随对应卡片内联，并说明它和我的判断是什么关系。
7. 不要修改 `{正式笔记路径}` 之外的任何文件。

## 迁移前回源预检

正式迁移前，必须先建立一份仅供内部核对的清单。清单不进入正式阅读笔记正文，但必须覆盖本轮全部拟处理材料。

| 稳定条目编号 | 章节 / range 线索 | 划线原文 | 我的原想法 | 外部读者原话 | 正式稿去向 | 处理动作 |
|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |

逐条预检时，必须确认：

1. 划线原文已经回源，不是凭印象补写，也不是只剩二手概括。
2. 我的原想法保留完整，没有被后续 AI 文字提前替换。
3. 章节或 range 线索足够支撑回源核对。若只是近似定位，要在内部清单中注明，正文里不要暴露技术痕迹。
4. 外部读者原话确有保留价值，不是重复感叹、鸡汤化表达或无法推进理解的情绪句。
5. 每条材料都已经决定正式稿去向，或已经明确写成“待回看 / 不迁移”的理由。
6. 不得伪造任何缺失材料。缺划线原文、缺原想法、缺位置依据时，先回到中间稿核对，不要硬迁移。

稳定条目编号、range、chapterUid、bookId、抓取状态等字段，只用于内部清单、回源核对和 QA，不默认写入正式阅读笔记正文。

## 正式阅读笔记结构

正式阅读笔记中的完整卡、主卡和核心卡，默认按下面的顺序归档。顺序不能乱，证据层不能丢：

```markdown
**【划线原文】**
> 原书划线原文。

**【我自己写的内容】**
> 我的原想法。

**【AI评价】**

**【AI修正】**

**【AI补充】**

**【外部读者原话｜关系：回声 / 补充 / 挑战 / 反向解释 / 独立精彩】**（如有）
> 外部读者原话。

- AI简评：
```

执行时必须遵守：

1. `划线原文` 必须先出现，`我自己写的内容` 必须紧跟其后。
2. `AI评价 / AI修正 / AI补充` 只能建立在前面的原文之上，不能覆盖、替代或吞掉它们。
3. 轻卡也要保留“划线原文 + 我自己写的内容”，只是后面的 AI 部分可以更短。
4. AI修正必须独立可读，并回到明确的文本细节，不能只整理语序。
5. 核心卡、主卡和完整卡如果保留“问题链升级”，必须补上“当前回答（沿原问题）”“当前回答（新增角度）”和必要的“后文待回看”。
6. 核心卡、人物主线卡和完整阅读卡，应至少保留一处贴文本的“可入笔记的文学表达”。

正式阅读笔记是长期保留文档，不是过程记录。迁移时禁止保留：中间稿编号、临时编号、“本条来自第 N 轮中间稿”、冗长归档说明，以及多条历史游标。

## 下次接着整理位置

正式阅读笔记文末只保留一条最新的 `下次接着整理位置`。

```markdown
下次接着整理位置：第X章，“某段引文开头……”之后
```

游标更新规则：

1. 第 N+1 轮迁移前，先删除第 N 轮旧的 `下次接着整理位置`。
2. 再迁移第 N+1 轮内容，并顺手处理前几轮已优化但尚未迁移的遗留内容。
3. 本轮迁移完成后，只在文末追加一条新的 `下次接着整理位置`。
4. 不要在正式阅读笔记里累积多条历史游标。

## 所有材料最终都要有归宿

中间整理稿只是临时工作台。第三阶段不是只迁移“精华”，而是把不同价值层级的材料放到合适位置。

- 主卡候选，进入重点卡、文章核心候选或人物主线卡。
- 完整卡候选，进入完整阅读卡。
- 轻卡，进入轻卡存档或阅读心流片段。
- 暂存素材，进入主题素材库、人物素材库或对照材料。
- 同位置其他书友高赞想法，优先随对应卡片内联；无法归卡但独立精彩的，进入全局“外部读者精彩想法”小节。
- 无法定位想法，进入散点想法、待回看，或给出明确的不迁移理由。

处理时不要出现“有价值但无去向”的悬空材料。轻卡、完整卡、主卡、暂存素材、外部读者材料，都必须被安置、待回看，或清楚说明为何不迁移。

## 外部读者材料迁移规则

第三阶段迁移同位置其他书友高赞想法时，优先随对应卡片内联保存。只有无法归入具体卡片、却仍能独立推进作品理解的材料，才进入正式阅读笔记中的全局“外部读者精彩想法”小节。不要把独立精彩的想法单独归入主题素材库或人物素材库。

外部读者材料与我的想法，内部可区分五种关系：

1. 外部读者回声，和我的想法形成同向关系。
2. 外部读者补充，补出我没展开的文本细节、心理机制或叙事安排。
3. 外部读者挑战，提出与我的判断不同但有文本依据的解释。
4. 反向解释，从相反立场重读同一细节，让问题更复杂。
5. 独立精彩外部视角，和我的想法没有直接关系，但它本身贴文本、有张力、有问题意识或有表达穿透力。

迁移要求：

1. 第五类不代表价值低。如果精彩，可以随卡内联，必要时也可以进入全局小节。
2. 不要为了归档方便，把第五类硬塞进我的原判断，也不要改写成“我的想法的补充”。
3. 外部读者原话必须按原话保存，不得概括成“有读者说”，不得改写为用户第一人称判断。
4. 可以在原话后附一两句 AI 简评，说明它精彩在哪里、打开了哪个文本角度。
5. 外部读者材料不是每张卡的必填项。和我的想法没有直接关系而独立精彩的材料，不强行解释，仍可进入全局小节。
6. 点赞数、热度或接口默认排序只作为发现信号，正式迁移仍以文学分析质量为准。

## 迁移后防回归 QA

迁移完成后，逐项完成以下 QA。任何一项不过，都不能把本轮迁移视为完成：

1. 本轮每个稳定条目编号，都有正式稿去向、待回看去向，或明确的不迁移理由。
2. 每个已迁移条目的划线原文都已进入正式稿，我自己写的原想法没有被 AI 改写覆盖。
3. 保留的外部读者材料仍是原话，优先随卡内联，没有被概括成“有读者说”，也没有被改写成用户判断。
4. 正式稿正文没有 `source ID`、`bookId`、`chapterUid`、`range`、点赞数、抓取状态、官方版定位或过程性编号等工作台痕迹。
5. AI修正能独立读懂，有明确文本落点，不依赖“这里”“上面”等模糊回指。
6. 轻卡仍然轻而有味，没有被拔高成完整卡，也没有被压成无味提纲。
7. 已迁移材料和迁移前回源预检清单逐条核对，没有遗漏、错位或无去向材料。

完成这一阶段后，如果用户没有明确要求下一阶段，就停在这里，不要继续做 `全书收束整合`，也不要提前建立 `文章素材索引`。
