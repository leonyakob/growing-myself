# 微信读书通用提示词，第四阶段，读完整本书后的全书收束整合

只有当我明确说“读完整本书后做全书收束整合”“全书收束整合”“按全书整理正式阅读笔记”之类的话时，才执行这一阶段。

第四阶段的标准模型是 **阅读现场档案 + 文章素材索引**。核心原则是：**卡片档案为底座，文章素材索引为上层导航**。

下文中的 `{中间稿路径}` 与 `{正式笔记路径}`，分别指继承块里列出的该书中间整理稿路径与正式阅读笔记路径。

## 1. 第三阶段与第四阶段的边界

- 第三阶段处理的是，本轮或前几轮已优化材料如何迁入正式阅读笔记。
- 第四阶段处理的是，整本书读完后，如何把已经有归宿的卡片、修订链、待回看项和文章方向收束成整书结构。
- 第三阶段的任务重心是迁移优化后的轮次材料。第四阶段的任务重心是全书收束整合。
- 本阶段不是全局重跑第三阶段。除非材料缺失、重复、语义错误、判断已被后文修正，或被明确标记为待重迁，否则不要整体重迁已有正式稿。
- 若正式稿已有合格卡片，默认补结构、补链接、补阅读轨迹，不重复搬运。
- 第四阶段要保留阅读过程，不把早期浅判断、误读、犹豫和后来的修正偷偷磨平。

## 2. 产物边界，内部整合台账与正式阅读笔记

- **内部整合台账** 只供整合、判重、回源和 QA 使用，可以出现 `source ID`、`range`、重复判定、`source status`、`archive disposition`、处理动作、来源版本、`QA state`、是否 external-only 等技术字段。
- **正式阅读笔记** 只保留读者可读的归档卡片和紧凑的文章导航，不写整行台账字段，不贴技术状态标签。
- 正式阅读笔记正文里的锚点必须是可读锚点，比如卡片标题、章节或场景名、引文短句、我的原句片段。`source ID`、`range` 不能作为读者可见的文章方向锚点，也不能出现在正式阅读笔记正文。
- 原始 `我自己写的内容` 必须保留，AI修正只能另写，不能覆盖原文。
- 外部读者原话若保留，必须保持原话身份，不能改写成我的判断。
- 正式稿里读者先看到的是可读结构，不是内部处理痕迹。技术判断留在内部整合台账，正文只留下能阅读、能回到文本、能看出阅读变化的部分。
- 正式稿里不得出现内部流程词或工程词，例如 `ledger`、`snapshot`、`task`、`wave`、`QA state`、`source status`、`archive disposition`、接口坐标、抓取状态、重复判定字段等。若必须表达内部核对结果，要改写成自然中文，例如“本轮没有额外确认需要局部修补或新增的卡”。

## 3. 先建多轴内部整合台账

全书收束时，先给每条材料建立 **多轴** 判断，不要只贴一个“完成/未完成”标签，也不要跳过材料状态，直接把一切压成文章提纲。

| 轴 | 允许值与用法 |
|---|---|
| `source status` | `已在正式笔记中，结构合格，只补文章索引` / `已在正式笔记中，但需补 readable evidence / 用户原句 / 外部原话` / `中间稿已优化，尚未迁移` / `中间稿与正式稿重复，需合并或建立修订链` / `正式稿旧判断已被后文修正，保留为阅读轨迹` / `仅外部读者材料，无用户卡片锚点` / `来源不足，待回看` |
| `archive disposition` | `保留为轻卡` / `保留为完整卡` / `提升为核心卡` / `合并入其他卡` / `归档不迁移` / `待回看` |
| `evidence role` | `主题证据` / `人物线证据` / `写法证据` / `阅读轨迹证据` |
| `article links` | 0 个、1 个或多个文章方向 |
| `trajectory flags` | `早期误读` / `后文修正` / `判断变化` |
| `external-reader relation` | `回声` / `补充` / `挑战` / `反向解释` / `独立精彩` / `丢弃并说明理由` |

处理顺序必须按这个台账走：先判 `source status`，再判 `archive disposition`，再判 `evidence role`、`article links`、`trajectory flags` 和 `external-reader relation`。不要跳过前两步，直接把材料压成“文章素材”或“最终结论”。

## 4. 一卡双归宿，与两层正式稿结构

`一卡双归宿` 的意思是，每张有价值的卡都先保留自己的档案身份，再决定是否被 0 个、1 个或多个文章方向引用。它可以只留在档案里，也可以同时服务多个写作方向。

两层结构如下：

1. **阅读现场档案**，负责保留或补齐卡片本体、修订链、必要的外部原话、轻卡身份、完整卡判断和核心卡层级。
2. **文章素材索引**，负责把可写方向、核心问题、关联卡片、原始触动、张力和证据缺口排成可导航结构。**文章素材索引只引用卡片，不吞掉卡片，也不得让文章素材索引替代卡片。**

`阅读现场档案` 不能只写成卡片入口目录。正式稿进入全书收束时，必须先用一小段概括本书各阅读轮次、阶段或批次中的问题推进：早期主要关注什么，随后哪些判断被修正，后期哪些主题变成主线，哪些轻触动仍保留为阅读现场。之后再列轻卡、完整卡、核心卡、外部读者材料等入口。这个小段只交代阅读过程和问题变化，不写成全书总评，也不复制原卡片正文。

文章素材索引里的常用引用形态：

```markdown
- 关联卡片：卡片标题 / 章节或场景 / 引文短句
- 原始触动：保留我的原句或原句片段
- 证据作用：它在这个文章方向中承担什么功能
- 缺口：还缺哪类文本证据或对照材料
```

## 5. 全书收束后的正式阅读笔记骨架

正式阅读笔记正文里，`阅读现场档案` 必须在 `文章素材索引` 前面。先归档，再导航。文章索引不能替代卡片，也不能把卡片正文吞进提纲。

```markdown
## 全书收束整合

### 一、阅读现场档案
> 保留或补齐卡片本体；已存在且合格的卡片不重复搬运。
> 先概括本书各阅读轮次、阶段或批次的问题推进，再列卡片入口；不要只写入口目录。

### 二、文章素材索引
> 只做导航，不吞掉卡片，不写成文章草稿。

| 文章方向 | 核心问题 | 关联卡片 readable anchor | 原始触动/用户原句 | 张力 | 可用证据 | 缺失证据 |
|---|---|---|---|---|---|---|

### 三、阅读轨迹与判断变化
> 只收束真正能说明阅读变化的误读、犹豫、修正，不机械罗列全部过程。

### 四、待回看 / 归档不迁移
> 简短记录原因；详细处理台账留在内部证据，不进入正式稿正文。
```

正式阅读笔记正文里，不出现 `source status`、`archive disposition`、`source ID`、`range`、抓取状态或 QA 状态。它们只留在内部整合台账。

全书收束完成后，若原笔记中仍有“下次接着整理位置”之类的阶段追踪语，应改为“全书整理完成位置”或类似完成态标题，说明已整理至终章，并转入全书回看、文章选题和专题扩写阶段。不要保留会让读者误以为整理尚未完成的追踪语。

## 6. 判重、回源与修订链规则

判重时先看来源关系，再看解读关系。

- 来源关系看四类：`同一引文` / `引文重叠` / `同一场景` / `不同引文`。
- 解读关系看五类：`同一判断` / `修正判断` / `不同角度` / `外部回声` / `外部挑战`。

处理原则：

1. `同一引文` 或 `引文重叠`，且属于 `同一判断`，选更清楚的一张做代表卡，另一张记入内部整合台账，`archive disposition` 记为 `合并入其他卡`。
2. `同一引文`，但属于 `修正判断`，不要硬并成一张最终卡，而要建立修订链。当前可用判断进 `阅读现场档案`，较早读法进 `阅读轨迹与判断变化`。
3. `同一场景` 但 `source ID` 或 `range` 不同，只要动作链和核心张力相同，就视为同一场景卡处理。正式稿使用可读场景名和引文短句，不用技术来源号做锚点。
4. `不同引文` 且属于 `不同角度`，只要承担的 `evidence role` 不同，就分别保留，不为了整齐硬合并。
5. `外部回声` 和 `外部挑战` 只能附着或照亮用户卡片，不能顶替用户卡片成为正式稿的主体。

## 7. 误读修正与阅读轨迹写法

当旧判断被后文修正时，不删旧卡，按以下结构收束：

- `当时的读法`
- `后来出现的证据`
- `全书后的修正`
- `误读的价值`
- `文章索引用法`

普通文章方向默认引用“全书后的修正”。只有当文章本身讨论阅读过程、判断如何变化、我如何被叙事带偏时，才把误读链作为 `阅读轨迹证据` 一并挂入文章素材索引。

## 8. 轻卡、外部读者材料与防膨胀规则

- 轻卡即使被文章素材索引引用，仍然可以保持轻卡，不自动升级为完整卡。它可以承担气氛证据、语言风格证据、主题回声或文章开头材料。
- 只有外部读者材料、没有用户卡片锚点的条目，先记入内部整合台账，`source status` 记为 `仅外部读者材料，无用户卡片锚点`。这类材料不能直接改写成我的第一人称判断，也不能直接充当文章索引的主锚点。
- 外部读者原话可以是 `回声`、`补充`、`挑战`、`反向解释` 或 `独立精彩`，但都不能覆盖我的原判断，不能把外部评论改写成“我其实一直这样想”。
- 鸡汤化、重复感叹、脱离文本太远的外部材料，可以记为 `丢弃并说明理由`，不要为了显得丰富硬塞进正式稿。
- 不是每张卡都必须进入正文卡片区。`归档不迁移` 和 `待回看` 都是有效去向。
- 为了防止正文膨胀，文章素材索引只写紧凑导航，不复制整张卡正文，不堆同义弱卡，不把内部判重理由整行贴进正式阅读笔记。
- 原始 `我自己写的内容` 不能被 AI修正覆盖。即便全书后判断改变，也是在下方补修正，不是回头抹掉当时的原句。
- 轻卡不因为后续可能写文章，就被强行扩成完整卡。外部读者材料不因为说得漂亮，就被抬成用户主判断。根 `AGENTS.md` 里的轻卡/主卡边界、文本优先和外部读者材料边界，在本阶段继续有效。

## 9. 执行顺序

1. 通读 `{正式笔记路径}` 和 `{中间稿路径}`，确认哪些卡已经合格，哪些只缺 readable evidence，哪些仍在中间稿里，哪些已经构成修订链。
2. 先搭内部整合台账，再决定哪些内容进入 `阅读现场档案`，哪些只进入 `文章素材索引`，哪些进入 `阅读轨迹与判断变化`，哪些进入 `待回看 / 归档不迁移`。
3. 若正式稿已有合格卡片，优先补结构、补链接、补阅读轨迹，不重复搬运整卡。
4. 若发现重复卡、改判卡、同场景异来源卡，先按判重与修订链规则处理，再写正式稿。
5. 写正式稿时，先写 `阅读现场档案`，再写 `文章素材索引`。不得反过来，也不得只留下文章方向，把卡片本体抽空。
6. 完成后做 fixture-based QA。QA 不通过，本阶段不得算完成。

## 10. fixture-based QA

完成前，必须对照 `.omo/evidence/fixtures/add-whole-book-consolidation-model.md` 做 dry-run。至少覆盖以下九类 fixture：

1. 部分迁移卡，只剩 AI 概括，需补 readable evidence 与用户原句。
2. 同引文、同判断的重复卡，需要选代表卡，其余并入内部台账。
3. 同引文、改判的卡，需要建立修订链，而不是偷换成单一最终结论。
4. 同场景、不同 `source ID/range` 的卡，若动作链与张力相同，按同一场景卡处理。
5. 早期误读、后期修正的卡，要保留阅读轨迹，不把误读当废料清空。
6. 外部高赞评论对用户卡形成挑战时，主体仍是用户卡，外部材料只作挑战或补充。
7. external-only 强评论，没有用户卡片锚点时，只能留在内部整合台账或 `待回看 / 归档不迁移`，不能直接充当正式稿主体。
8. 被文章引用的轻卡，仍可保持轻卡，不自动升级成完整卡。
9. 为防正文膨胀而作出的 `归档不迁移` 处理，必须写明原因，但原因留在可读结构边界内，不把整行内部判重字段贴进正文。

至少逐项确认：

1. 正式稿里先有 `阅读现场档案`，后有 `文章素材索引`。
2. `文章素材索引只引用卡片，不吞掉卡片`，没有把整张卡正文复制成文章提纲。
3. 正式稿正文和文章方向锚点里没有 `source ID`、`range` 等技术字段。
4. 轻卡没有因为被文章引用就被强行升级成完整卡。
5. 外部读者原话没有被改写成我的判断。
6. `我自己写的内容` 没有被 AI修正覆盖。
7. 有价值但不进正文的材料，已经在 `待回看 / 归档不迁移` 或内部整合台账中有明确去向。
8. `阅读现场档案` 开头已经交代阅读轮次、阶段或批次中的问题推进，而不是只列卡片入口。
9. 全书完成后的尾部追踪语已经切换为完成态，没有残留“下次接着整理位置”造成未完成感。
10. 正式稿正文没有残留 `ledger`、`snapshot`、`task`、`wave` 等内部流程词或工程词。

## 11. 完成标准

满足以下条件，才算第四阶段完成：

- 正式阅读笔记呈现出 `阅读现场档案 + 文章素材索引` 的两层结构。
- `阅读现场档案` 在前，`文章素材索引` 在后。
- 正文中保住卡片本体、用户原句、必要时保留的外部原话身份，且不露出技术字段。
- 内部整合台账已经处理完判重、回源、修订链和 `归档不迁移` 去向。
- fixture-based QA 已完成，九类 case 都能被当前提示词覆盖。
- 没有把本阶段偷写成第三阶段补迁移，也没有把文章导航误写成卡片替身。
