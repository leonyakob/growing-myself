## Non-quota decision rule

- **There is no core-card maximum.** 文章核心候选数量不设上限，不能因为当前正式稿已有 5 张核心卡而压掉 `030` 或其他可独立成文的材料。
- **There is no complete-card maximum.** 完整阅读卡数量不设上限，不能因为正式稿整洁、篇幅、卡片数量或人物主线已存在，就把能支撑独立问题链的材料降成轻卡或素材库。
- 中间稿已经标为 `主卡候选` 或 `完整卡候选`，并且 Todo 1 ledger 显示有 `可扩写方向` 的条目，本轮默认提升到 `article-core candidate` 或 `complete reading card`；除非文本本身不能成立独立问题链，才允许下调。本表最终没有把任何 `主卡候选` / `完整卡候选` 下调为轻卡、人物主线或素材库。
- 外部读者材料的有无不单独决定强弱：`006` 为 `unclear` 仍定为完整阅读卡；`076、103、104` 为 `no` 仍定为完整阅读卡；`073、113` 为轻卡，是因为文本功能较短、不是因为没有外部材料。
