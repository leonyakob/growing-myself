# Cultural Aesthetic Brand Training Scheme Plan

## TL;DR
> **Summary**: Produce the final long-term training scheme for a future high-end Eastern life aesthetics founder, correcting the prior failure by centering cultural literacy, personal essence extraction, and cumulative AI style learning before aesthetics and brand systems. The final scheme must be organized by ability modules first, then a 30/90/180-day training rhythm for 10-20 minute medium-long videos.
> **Deliverables**:
> - `.omo/drafts/cultural-aesthetic-brand-training-scheme.md`: final training方案 in Chinese
> - A complete ability-module skeleton with detailed exercises
> - A 30/90/180-day training calendar matched to the user's time budget
> - Resource system covering books, videos, blogs, lectures, documentaries, museums, and examples
> - Templates for essence extraction, modern translation, object/space observation, AI style learning, and 10-20 minute video scripts
> - Verification rubric proving the scheme answers the user's critique
> **Effort**: Medium
> **Parallel**: YES - 4 waves
> **Critical Path**: Tasks 1-4 -> Tasks 5-8 -> Tasks 9-10 -> Task 11 -> Final Verification Wave

## Context

### Original Request
The user asked how to train three capabilities:

1. 东方文化的现代转译能力
2. 稳定统一的审美表达能力
3. 品牌化的系统能力

The user rejected the previous answer because it was too focused on visual aesthetics and brand packaging while missing the core need: cultural literacy, personal interpretation, and personally meaningful “精华” that can be injected into AI-assisted medium-long video scripts.

### Interview Summary
- Final output should be a detailed training方案, but this plan is the execution plan for creating that方案.
- The final scheme must combine both structures:
  - ability modules first
  - then 30/90/180-day training rhythm
- “精华” must be broad and open-ended, not limited to age-stage rereading, agreement/disagreement, epiphany, or misunderstanding-to-understanding. Those were only examples.
- AI collaboration must be cumulative style learning through repeated dialogue, style samples, corrections, and revision history. It must not be a one-off prompt library.
- Main output format is 中长视频, not short video.
- Target video length is 10-20 minutes.
- Training time budget:
  - Month 1: 3+ hours/day
  - Months 2-3: about 2 hours/day
  - After 3 months: about 1 hour/day
- Persona: 东方生活美学主理人, with high knowledge, restraint, affection, judgment, brightness, gentleness, and warmth. Avoid heavy, cold, overly academic, or solemn tone.
- First 90 days must prioritize 文学文化 + 器物空间.

### Metis Review (gaps addressed)
- Guardrail: create an execution plan, not the full training方案 directly.
- Guardrail: visual identity and brand packaging must remain downstream of cultural reading and personal interpretation.
- Guardrail: define “精华” as broad personal meaning extraction, not a rigid worksheet.
- Guardrail: AI workflow must include cumulative style assets, not only prompts.
- Guardrail: output practice must be for 10-20 minute medium-long video, not short video logic.
- Guardrail: brand-system work must not become logo/color/account packaging or premature monetization architecture.
- Guardrail: the final scheme must explicitly explain how it corrects the previous failure.

### Oracle Phase 1
Oracle returned `VERDICT: GO`: interview context is sufficient to generate this execution plan without further user questions.

### Oracle Phase 2
Oracle initially returned `VERDICT: NO-GO` because the plan still contained human-approval wording, vague QA terms, underspecified resource counts, unenforceable guardrails, and underspecified final verification tasks. The plan was corrected by adding deterministic rubrics, agent-only PASS/FAIL final verification, exact resource bounds, forbidden-term checks, and full F1-F4 verification blocks. Oracle then returned `VERDICT: GO`.

### Deterministic Verification Rubric
> Every pass/fail decision must be machine-checkable or countable by the executor.

1. **Section order rubric for the final scheme**
   Required top-level order:
   1) 用户诊断 / 需求纠偏
   2) 核心训练哲学
   3) 能力模块
   4) 训练 loops
   5) 资源系统
   6) AI 累积式风格学习
   7) 10-20 分钟中长视频系统
   8) 30/90/180 天节奏
   9) 模板
   10) 验证 rubric
   11) Must Not Have / 负面约束

2. **Module rubric**
   Every module must include exactly these fields:
   - 目的
   - 训练能力
   - 每日练习
   - 每周练习
   - 输出物
   - 复盘标准
   - 常见失误
   - 与 10-20 分钟中长视频的关系

3. **Resource rubric**
   The resource system must contain exactly 7 top-level categories:
   - 根文本
   - 解释桥梁
   - 具身文化
   - 现代转译资源
   - 当代文化批评
   - 高端/奢侈意义建构
   - 输出写作工艺
   Each category must include 3-5 priority resource clusters or items, and each cluster/item must have these fields:
   - 名称
   - 类型
   - 训练能力
   - 为什么推荐
   - 如何使用
   - 提取什么精华
   - 可产出什么内容物
   The resource system must include at least 5 non-book source types in total from this set: 视频, 纪录片, 讲座, 博客/专栏, 博物馆/展览/图录, 访谈, 田野观察.
   Premium/brand meaning-making resources must be no more than 20% of total clusters and must appear after the cultural-root and interpretive-bridge categories.

4. **AI workflow rubric**
   Must include all of these assets:
   - personal insight archive
   - style sample archive
   - preferred tone card
   - forbidden tone card
   - correction log
   - before/after rewrite log
   - persona calibration prompt
   - periodic output audit
   - style rule update mechanism

5. **Medium-long video rubric**
   Each script system must explicitly support 10-20 minute videos and include these blocks:
   - thesis opening
   - cultural/literary/object scene
   - interpretive turn
   - personal essence moment
   - modern translation
   - aesthetic landing
   - soft CTA
   - segment timing
   - retention without clickbait

6. **Forbidden-term rubric**
   The final scheme must include a forbidden-term list, and each forbidden term must appear only in the Must Not Have / Guardrail sections, not in positive recommendation sections.
   Forbidden terms to audit: 爆款, 涨粉, 算法, 短视频钩子, logo, 配色, 产品阶梯, 付费社群, 变现.

7. **Ordering rubric**
   In the final scheme, the first occurrence of cultural reading / 精华 must come before aesthetics, before AI/video, and before brand/channel assets.
