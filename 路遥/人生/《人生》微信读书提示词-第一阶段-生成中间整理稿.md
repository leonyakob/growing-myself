整理《人生》微信读书笔记时，默认以“我的想法”为主索引整理，优先处理有我的想法的条目。只有划线、没有我的想法的条目暂不整理，除非我明确要求整理纯划线。

这里的“以我的想法为主索引”，不是只整理我的想法文本。必须同时获取两路数据，用划线接口补足原文、颜色、章节和位置：

## 4. 数据抓取与匹配规则

### 4.1 个人数据主接口

1. `/book/bookmarklist`：抓个人导入版划线原文、章节、位置、颜色 `colorStyle`。
2. `/review/list/mine`：抓个人导入版我的想法、`abstract`、`chapterUid`、`range`。

### 4.2 划线与想法匹配顺序

不要因为 `bookmarklist` 和 `review/list/mine` 的 `range` 不完全一致，就直接判断为“只有想法”或“无法匹配”。跨段划线、段尾想法、第二段想法都可能导致 `range` 不完全相同。

匹配顺序：

1. **精确匹配**：`bookId + chapterUid + range` 完全一致。
2. **跨段匹配**：`bookId` 和 `chapterUid` 相同，想法 `range` 落在某条跨段划线 `range` 内。
3. **重叠匹配**：`bookId` 和 `chapterUid` 相同，想法 `range` 与划线 `range` 有重叠或相邻，且文本内容连续。
4. **文本判断匹配**：想法内容明显对应某条划线原文，即使 `range` 不完全一致，也可合并，但必须标注“非精确 range 匹配”。

输出时每条都要保留：

1. 匹配方式：精确 range / 跨段 range / 重叠 range / 文本判断匹配 / 未匹配。
2. 是否需要人工确认：是 / 否。
3. 后续处理方向。

效率边界：

1. 不要为了少量未精确匹配条目反复重拉全书、重整旧材料。
2. 未匹配但有想法的条目先列入“有想法但划线匹配待确认”，不要自动扩大整理范围。
3. 只有划线、没有想法的条目，本轮不整理，除非我明确要求整理纯划线。
4. 如果某条想法没有 `range`，或按以上规则仍无法和划线原文稳定合并，应单独放入“有想法但划线匹配待确认”小节，不要强行归并，也不要因此重整旧材料。

### 4.3 每条整理材料至少保留的信息

1. 章节名
2. `chapterUid`
3. `range`
4. 匹配方式
5. 是否需要人工确认
6. 划线颜色
7. 划线原文
8. 我的原想法
9. 同位置其他书友高赞想法抓取状态：已获取并保留 / 已获取但未保留 / 未能定位官方版同位置 / 同位置暂无公开想法 / 接口错误
10. 官方版定位：`bookId / chapterUid / range`，若可定位
11. 同位置其他书友高赞想法摘录：按卡片价值保留 `0-5` 条
12. 与我的想法的初步关系：回声 / 补充 / 挑战 / 反向解释 / 独立精彩 / 暂无有效高赞想法
13. 想法类别
14. 卡片判断
15. 后续处理方向

## 5. 导入版到官方公开版的同位置映射工作流

整理《人生》本轮新增内容时，默认以“我写了想法的位置”为索引，尝试查询同一文本位置下其他书友的公开高赞想法。

术语定义：

1. **同位置其他书友高赞想法**：指我在某段文本 A 上划线并写下个人想法后，其他书友也在同一段文本 A 或高度重叠文本上发表的公开想法中，点赞数相对靠前，且能对文本理解形成回声、补充、挑战、反向解释或独立启发的内容。
2. **同位置**：优先按文本内容对应判断，不按个人导入版 `bookId + chapterUid + range` 直接相同判断。若个人笔记来自导入书，应先将导入版坐标映射到微信读书官方版坐标。
3. **高赞**：以 `/book/readreviews` 返回的 `likesCount` 为主要发现信号，但不以点赞数作为唯一收录标准。

已知《人生》存在两个 `bookId` 体系：

1. 个人导入版：`CB_2tb79r78T38k74M75h8iz4C3`，用于抓取我的划线、颜色和个人想法。
2. 微信读书官方公开版：`25164497`，用于抓取公开热门划线和同位置其他书友公开想法。

因此，不能直接用个人导入版 `CB_2tb79r78T38k74M75h8iz4C3 + chapterUid + range` 判定公开想法是否存在。正确流程是：先用个人导入版整理我的阅读现场，再按章节标题、划线原文和文本重叠映射到官方版 `25164497` 的同一文本位置，最后用官方版 `bookId + chapterUid + range` 调 `/book/readreviews`。

### 5.1 接口分工

1. `/book/bookmarklist`：抓个人导入版划线原文、章节、range、颜色 `colorStyle`。
2. `/review/list/mine`：抓个人导入版我的想法、`abstract`、`chapterUid`、`range`。
3. `/book/bestbookmarks`：抓官方版热门划线，含官方版 range、划线文本和划线人数。它用于文本匹配和公开版定位，但每章最多返回 20 条，不覆盖全部位置。
4. `/book/underlines`：抓官方版章节内划线热度 range、count、score，但不含文本。它只能作为辅助信号，不能单独证明文本同位置。
5. `/book/readreviews`：用官方版 `bookId + chapterUid + range` 抓该划线下其他书友公开想法。
6. `/review/list`：是整本书公开点评 / 公开想法列表，不能按我的章节和 range 定位同一划线位置，不作为同位置高赞想法主接口。

### 5.2 官方公开版同位置匹配优先级

1. **精确文本匹配**：我的划线原文和官方版热门划线原文完全一致。
2. **包含匹配**：我的划线原文是官方版划线的一部分，或官方版划线是我的划线的一部分。
3. **高重叠匹配**：两段文字大部分重叠，只是前后多了一两句。
4. **章节内近似文本匹配**：章节标题相同，核心句子相同，但 `range`、段落边界不同。
5. **无法稳定匹配**：找不到足够相同的公开版文本，才标“未能定位官方版同位置”。不能因为我的个人 `range` 没出现在热门划线列表里，就直接判定失败。

### 5.3 候选池与保留数量

1. 所有能稳定定位官方版同位置的条目，默认至少尝试抓取一批 `/book/readreviews` 公开想法，通常 `count=20`。
2. 轻卡默认保留 `0-1` 条。只有当一条外部高赞想法能明显照亮我的触动、形成挑战或补充时才保留。
3. 完整卡候选默认保留 `1-3` 条。如果高赞想法出现多个不同解释方向，应优先保留差异，而不是只保留点赞最高的一条。
4. 主卡候选 / 文章核心候选默认保留 `2-5` 条。必要时可继续翻页扩大候选池，例如最多查看 40 条，但必须服务于解释分歧、主题张力或问题链推进。
5. 保留多条时必须说明每条分工：回声 / 补充 / 挑战 / 反向解释 / 独立精彩。
6. 同质重复的外部高赞想法只保留 1 条代表。方向相反但都贴文本的外部高赞想法应优先并存。

### 5.4 筛选规则与输出状态

筛选规则：

1. 按 `likesCount` 发现候选，但不要机械取最高赞。
2. 先过滤纯骂人、纯表情、纯感叹、复读原文、鸡汤化、名言堆砌但不贴文本、离文本太远、只做道德审判但不能推进理解的内容。
3. 再按观点方向分簇：回声、补充、挑战、反向解释、独立精彩、暂不采用。
4. 每个观点簇里优先选点赞较高、贴文本、表达清楚、能推进我的问题链的想法。
5. 若同一划线下出现多条高赞想法，且方向相反但都能贴文本解释，应标为“外部读者分歧”，不要只保留更赞同我的一条。

输出状态必须区分：

1. **已获取并保留**：已定位官方版同位置，并筛出有效高赞想法。
2. **已获取但未保留**：接口有外部想法，但多为低质量、同质重复或不能推进文本分析。
3. **未能定位官方版同位置**：个人导入版坐标与官方公开版坐标不同，当前未能通过原文匹配找到对应官方版 range。
4. **同位置暂无公开想法**：已定位官方版同位置并调用 `/book/readreviews`，但 `totalCount` 为 0。
5. **接口错误**：调用失败、鉴权失败、服务端错误等，应记录错误码或简要原因。

高赞想法只能作为外部读者材料，不得替代我的感受、问题和判断。

## 6. 颜色默认流向

微信读书划线颜色通过 `colorStyle` 字段识别。颜色只作为第一阶段的轻量预分类参考，不是最终卡片类型。

| colorStyle | 颜色名称 | 颜色默认流向 |
|---|---|---|
| `1` | 粉色 / 红色 | 轻卡，记录气氛、心动、暂不深挖的触动 |
| `2` | 紫色 | 人物卡，记录人物心理、关系、命运选择 |
| `3` | 蓝色 | 主题卡，记录阶层、城乡、贫穷、劳动、尊严、现实结构 |
| `4` | 青色 | 细读卡，记录写法、场景、动作、物件、空间、结构、沉默 |
| `5` | 黄色 | 对照卡，记录跨作品、同书前后、作者生平、现实经验联动 |
| `0` | 未分类 | 未分类旧材料 |

输出时不要把颜色写成单纯数字。正文里写“蓝色（主题卡默认流向）”，不要只写 `colorStyle：3`。

整理时必须保持阅读顺序，不要为了按颜色分类而打散阅读线索。颜色统计和主卡候选可以放在文末汇总。

## 7. 想法类别

“想法类别”指我在微信读书里写下的想法本身是什么形态。

1. **感受型**：情绪、心动、惊讶、喜欢、心疼、厌恶、共鸣等，可能很短，但要保留。
2. **浅判断型**：已经有判断，但证据、问题链、文本分析还不充分。
3. **问题型**：主要提出疑问，暂时没有充分判断。
4. **片段分析型**：已经开始分析人物、情节、动作或主题，但还不完整。
5. **完整分析型**：已经接近正式阅读笔记里的完整写法，包含“触动点 / 为什么停下来 / 问题 / 判断 / 扩写方向”等要素，或虽然没显式分栏但分析已经比较完整。
6. **素材型**：主要是为以后文章保存证据、句子、对照材料，当前不急着分析。

## 8. 卡片判断

“卡片判断”指这条材料后续怎样处理。本阶段只做轻量预分类，不做第二阶段深度优化。

1. **轻卡**：保留触动、感受、气氛和直觉，不强行深挖。
2. **完整卡候选**：有明显人物心理、动作细节、结构反常、主题张力，可以补成完整阅读卡。
3. **主卡候选**：不仅能补成完整卡，而且可能发展成文章核心、章节核心或全书母题。
4. **暂存素材**：目前主要作为证据、对照材料或主题材料保存，暂不做完整评价。

轻卡和完整卡不是按字数机械区分，而是看它是否已经形成可追问的矛盾和证据链。
