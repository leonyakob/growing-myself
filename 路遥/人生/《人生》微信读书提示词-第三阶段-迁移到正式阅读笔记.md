只有当用户明确说“把已优化内容更新到正式阅读笔记”，或明确要求把已优化内容整理并更新到《人生》阅读笔记里，才执行这一阶段。用户如果只是要求继续整理、继续优化、只评价问题质量、回看中间稿，或只是咨询流程，不要进入本阶段。

## 固定工作原则

1. 不要机械搬运所有中间稿，要按材料价值分类安置。
2. 默认优先迁移本轮已优化内容，同时检查前几轮已优化但尚未迁移的遗留内容。有价值的应一并迁移，或明确列入待回看，不要默认遗漏。
3. 每条迁移材料都必须回到源原文，保留书籍划线原文与我的原想法。正式阅读笔记采用 `划线原文 / 我自己写的内容 / AI评价 / AI修正 / AI补充` 这条主轴，AI 内容只能建立在原文之上，不能替代原文。
4. `我自己写的内容` 栏必须完整保留我的原想法。允许极少量错别字修正，但不允许把 AI修正 填进这一栏，也不允许用 AI修正 抹掉原来的表达。
5. 轻卡可以轻，完整卡与主卡要更完整，但任何材料都不能失去原始划线与原始想法。
6. 正式阅读笔记是长期保留文档，不是工作台。迁移时要去掉中间稿痕迹、过程编号与接口字段痕迹。

## 正式阅读笔记结构

完整卡、主卡、核心卡以及需要沉淀的轻卡，都优先按以下结构写入正式阅读笔记：

```markdown
**【划线原文】**
> 原书划线原文。

**【我自己写的内容】**
> 我的原想法。

**【AI评价】**

**【AI修正】**

**【AI补充】**

> 外部读者回声 / 补充 / 挑战 / 反向解释 / 独立精彩：“外部读者原话”
> 简评：
```

结构要求：

1. 合并卡可以把多个材料放在同一组，但每个材料仍须保留自己的 `划线原文 + 我自己写的内容`。
2. 轻卡也保留相同结构，只是 `AI评价 / AI修正 / AI补充` 可以简短，不必硬拉出完整卡的问题链负担。
3. `AI修正` 应独立可读，并回到明确文本细节，不能靠“这里”“上面”“这一段”这类模糊回指撑住。
4. 如果保留问题链升级，必须补出“当前回答（沿原问题）”“当前回答（新增角度）”和必要的“后文待回看”。
5. 核心卡、人物主线卡和完整阅读卡应有贴文本的文学表达，但文学表达可以自然融入 `AI修正`、`AI补充` 或标题，不强制另立字段。
6. 外部读者原话是可选项，不是每张卡的必填栏。只有它确实推进文本理解时才保留。
7. 不要把结构写成 `【源ID：035｜划线原文】` 这种工作台格式。

正式阅读笔记正文禁止保留：

- 中间稿编号
- 临时编号
- “本条来自第 N 轮中间稿”
- 冗长归档说明
- 多条历史“下次接着整理位置”
- `source ID`
- `bookId`
- `chapterUid`
- `range`
- 点赞数
- 抓取状态
- 官方版定位信息
- 其他接口字段或回源字段

## 正式阅读笔记游标更新规则

每次更新正式阅读笔记时，文末只保留一条最新的“下次接着整理位置”。

执行顺序：

1. 先删除旧的“下次接着整理位置”。
2. 再迁移本轮内容，并顺手处理前几轮已优化但尚未迁移的遗留内容。
3. 迁移完成后，在文末追加一条新的“下次接着整理位置”。

不要在正式阅读笔记里累积多条历史游标。

```markdown
下次接着整理位置：第X章某段文字之后
```

## 外部读者精彩高赞想法迁移规则

第三阶段迁移高赞想法时，优先随对应卡片内联保存。只有无法归入具体卡片、却仍能独立推进《人生》理解的材料，才进入正式阅读笔记中的全局“外部读者精彩高赞想法”小节。不要把独立精彩的想法单独塞进主题素材库或人物素材库。

关系标签固定区分五类：

1. **回声**，与我的想法同向，强化原触点。
2. **补充**，补出我的原判断没有展开的角度。
3. **挑战**，反驳、纠偏或迫使我收稳原判断。
4. **反向解释**，从另一套逻辑解释同一文本，与我的想法形成并列张力。
5. **独立精彩外部视角**，与我的想法无直接关系，但原话本身足够贴住《人生》的文本细节，有张力、有问题意识，或有表达穿透力。

迁移要求：

1. 第五类只是不形成回声，不代表价值低。如果精彩，应和其他类型一起进入“外部读者精彩高赞想法”。
2. 不要为了归档方便，把独立精彩外部视角硬塞进我的原判断，也不要改写成“我的想法的补充”。
3. 外部读者原话必须以原话保存，不能概括成“有读者说”。如需说明关系，可在原话后附一两句 AI 简评。
4. 外部读者材料不是每张卡的必填项。与我的想法没有直接关系而独立精彩的材料，不强行解释，仍可进入全局小节。
5. 点赞数、热度或接口默认排序只作为发现信号，是否迁移仍以文学分析质量和文本推进力为准。
